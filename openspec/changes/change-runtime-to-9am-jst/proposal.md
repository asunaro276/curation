# 変更: 実行時間を日本時間午前9時に変更

## なぜ
現在、自動実行は UTC 9:00（日本時間18:00）に設定されており、夕方に技術ニュースが配信されています。しかし、ユーザーが朝の業務開始時に最新の技術情報を確認できるよう、実行時間を日本時間の午前9時（UTC 0:00）に変更する必要があります。朝の時間帯に配信することで、ユーザーは1日の始まりに最新のテクノロジートレンドをキャッチアップでき、より有効に情報を活用できます。

## 何が変わるか
- GitHub Actions ワークフローの cron スケジュールを `0 9 * * *` (UTC 9:00) から `0 0 * * *` (UTC 0:00) に変更
- 実行時間が日本時間で午後6時から午前9時に変更される
- ワークフロー内のコメントを新しい実行時間に合わせて更新

## 影響
- 影響を受ける仕様: `scheduled-execution`
- 影響を受けるコード: `.github/workflows/run-curation.yml`
- ユーザーへの影響: 技術ニュースの配信時間が夕方から朝に変更される
- 運用への影響: なし（単純な時刻変更のみ）
