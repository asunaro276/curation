# プロジェクト コンテキスト

## 目的
技術記事、ライブラリのリリースノート、プレスリリースを自動収集し、LLMで要約してSlackに定期投稿する自動キュレーションシステム

## 技術スタック
- Ruby (最新版)
- GitHub Actions (定期実行基盤)
- Claude API (Anthropic) - 記事要約用
- Slack Webhook - 投稿配信用

## プロジェクトの規約

### コードスタイル
- RuboCop準拠
- Ruby標準のスタイルガイドに従う
- 明確なクラス/モジュール構成

### アーキテクチャパターン
- モジュール化されたコンポーネント設計
- 収集、要約、配信の責任分離
- 設定ファイルベースの管理
- 依存性注入パターンの使用

### テスト戦略
- RSpec使用
- ユニットテストとインテグレーションテスト
- 外部API呼び出しのモック化
- CI/CDパイプラインでの自動テスト実行

### Git ワークフロー
- main ブランチ保護
- feature/* ブランチで開発
- コミットメッセージは日本語可

## ドメインコンテキスト
- 技術ニュースソース: Hacker News, GitHub Trending, 技術ブログRSS等
- 情報の鮮度が重要（最新の技術トレンドを追跡）
- 要約品質はユーザー体験に直結
- 投稿頻度は設定可能（日次、週次等）

## 重要な制約
- GitHub Actions無料枠内での実行時間制限
- Claude API利用量・コスト管理
- Slack投稿のレート制限遵守
- 環境変数での機密情報管理必須

## 外部依存関係
- Claude API (Anthropic) - 要約生成
- Slack Webhook API - メッセージ投稿
- 各種技術ニュースソース (RSS, API等)
- GitHub Actions - 実行環境
