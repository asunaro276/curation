# Capability: ニュース収集 (news-collection)

## 説明
複数の技術情報ソース（RSS、API等）から記事データを自動収集する機能

## ADDED Requirements

### Requirement: 複数ソースからの収集サポート
システムは複数の技術ニュースソース（RSS、API等）から記事を収集できなければならない (MUST)。

#### Scenario: RSSフィードからの記事収集
**Given** 有効なRSSフィードURLが設定されている
**When** 収集プロセスが実行される
**Then** 最新の記事がパースされて取得される
**And** 各記事のタイトル、URL、公開日時、説明が抽出される

#### Scenario: GitHub Trending APIからの取得
**Given** GitHub APIアクセスが設定されている
**When** トレンドリポジトリ収集が実行される
**Then** 人気上昇中のリポジトリ情報が取得される
**And** リポジトリ名、説明、スター数、言語が含まれる

### Requirement: 設定可能なソース管理
ニュースソースは設定ファイルで管理され、容易に追加・削除できなければならない (MUST)。

#### Scenario: 設定ファイルからのソース読み込み
**Given** `config/sources.yml` にニュースソースが定義されている
**When** システムが起動する
**Then** 全ての設定されたソースが読み込まれる
**And** 無効または不正な設定はログに警告が記録される

### Requirement: エラーハンドリング
個別のソースでエラーが発生しても、他のソースの収集は継続されなければならない (MUST)。

#### Scenario: 一部ソースの取得失敗
**Given** 3つのニュースソースが設定されている
**When** 1つのソースがタイムアウトまたはエラーを返す
**Then** エラーはログに記録される
**And** 残りの2つのソースからは正常にデータが収集される
**And** 処理は中断されない

### Requirement: レート制限の遵守
外部APIにアクセスする際、レート制限を尊重し、適切な間隔でリクエストしなければならない (MUST)。

#### Scenario: API呼び出し間隔の制御
**Given** APIレート制限が設定されている（例: 1秒あたり1リクエスト）
**When** 複数のAPIエンドポイントに連続してアクセスする
**Then** リクエスト間に適切な待機時間が挿入される
**And** レート制限エラーが発生しない
